Для просмотра страницы была использована следующая ссылка:
http://localhost:8080/students

Приложение предоставляет веб-интерфейс, реализованный на Thymeleaf.
При переходе по адресу /students отображается страница "Список студентов".

Перед просмотром страницы Spring Security перенаправляет на /login
с формой "Please sign in" (с полями username и password).

Что имеется на самой странице:
1. Форма добавления студента: поля "Имя" и "Фамилия" + кнопка "Добавить"
2. Таблица с колонками: ID, Имя, Фамилия, Действия
3. Кнопка "Удалить" напротив каждой записи
4. Кнопка "Выйти" и подсказка с логинами

В БД были вот такие данные (айди - имя - фамилия):
1 - Sash - Brown
2 - Jone - Smith
3 - Deema - Johnson
4 - Ekatrena - Wilson
5 - Sash - Brown

Доступные учётные записи:
- user / password  — только просмотр списка
- admin / password — просмотр + добавление + удаление

При попытке удалить/добавить студента, залогинившись под user:password,
приложение вернёт Whitelabel Error Page:
  status=403, error=Forbidden
Это подтверждает, что Spring Security корректно ограничивает доступ
к операциям удаления для роли USER.

Пользователь с ролью ADMIN может выполнять все CRUD-операции через UI.
