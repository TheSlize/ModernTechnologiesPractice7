# Часть 4: Thymeleaf

1. Что делает `th:object`?

Ответ: `th:object` привязывает HTML-форму к конкретному Java-объекту из модели. В шаблоне `students.html` написано `th:object="${student}"` — это означает, что форма работает с объектом `student`, который контроллер добавил в модель через `model.addAttribute("student", new Student())`. Все поля внутри формы могут обращаться к этому объекту через сокращённый синтаксис `*{...}` вместо полного `${student.name}`.

2. Как `th:field="*{name}"` связывает поле формы с объектом?

Ответ: `th:field="*{name}"` автоматически генерирует атрибуты `id="name"`, `name="name"` и `value="${student.name}"` для HTML-элемента `<input>`. Звёздочка `*{...}` — это сокращение, работающее в контексте `th:object`: оно обращается к полю `name` привязанного объекта `student`. При отправке формы браузер передаст значение поля с именем `name`, и Spring MVC через `@ModelAttribute` автоматически смапит его обратно в поле объекта `Student`.

3. Что означает `redirect:` в возвращаемой строке контроллера?

Ответ: `redirect:` — это специальный префикс Spring MVC, который указывает, что вместо рендеринга шаблона нужно отправить браузеру HTTP-ответ `302 Found` с заголовком `Location`. Браузер автоматически выполнит новый GET-запрос по указанному URL. В `StudentWebController` методы `save` и `delete` возвращают `"redirect:/students"` — это предотвращает повторную отправку формы при обновлении страницы (паттерн Post/Redirect/Get).
