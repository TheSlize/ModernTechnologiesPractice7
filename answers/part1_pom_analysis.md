# Часть 1: Создание проекта и анализ pom.xml

1. Что находится в секции `<parent>` и зачем она нужна?

Ответ: В секции `<parent>` указан `spring-boot-starter-parent` версии `3.5.0`. Это родительский POM от Spring Boot, который централизованно управляет версиями всех официальных зависимостей и плагинов через `dependencyManagement` и `pluginManagement`. Благодаря ему дочерний проект наследует предустановленные, совместимые между собой версии библиотек, настройки компилятора (Java 21), кодировку UTF-8 и конфигурацию плагинов по умолчанию — без необходимости дублировать эти настройки вручную.

2. Почему у большинства зависимостей нет версий?

Ответ: Потому что версии уже заданы в `spring-boot-starter-parent` (точнее, в его предке `spring-boot-dependencies`), который содержит раздел `<dependencyManagement>` со списком всех совместимых версий зависимостей. Maven автоматически подбирает нужную версию из этого BOM (Bill of Materials), поэтому явно указывать версию в дочернем POM не нужно. Это гарантирует совместимость всех библиотек друг с другом и упрощает обновление — достаточно поднять версию родителя.

3. Что делает `spring-boot-maven-plugin`?

Ответ: `spring-boot-maven-plugin` выполняет две ключевые функции:
- **При сборке (`mvn package`)** — перепаковывает стандартный JAR в так называемый «fat JAR» (executable uber-JAR), включая внутрь все транзитивные зависимости и встроенный сервер (Tomcat). Это позволяет запускать приложение командой `java -jar app.jar` без установки внешнего сервера приложений.
- **При запуске (`mvn spring-boot:run`)** — запускает приложение прямо из Maven в режиме разработки, подхватывая classpath и `devtools` для горячей перезагрузки.

4. Какие транзитивные зависимости подключает `spring-boot-starter-web`? Выполните `mvn dependency:tree` и опишите 3–5 самых важных.

Ответ: `spring-boot-starter-web` подключает следующие ключевые транзитивные зависимости:

- **`spring-webmvc`** — ядро Spring MVC: DispatcherServlet, обработка HTTP-запросов, маппинг контроллеров (`@RequestMapping`, `@RestController`).
- **`spring-boot-starter-tomcat` → `tomcat-embed-core`** — встроенный сервер Tomcat, который позволяет запускать приложение как самостоятельный процесс без внешнего контейнера сервлетов.
- **`jackson-databind`** (через `spring-boot-starter-json`) — библиотека Jackson для автоматической сериализации/десериализации Java-объектов в JSON и обратно (используется в REST API).
- **`spring-context`** — контекст приложения Spring: IoC-контейнер, управление бинами, ApplicationContext и события жизненного цикла.
- **`spring-boot-autoconfigure`** (через `spring-boot-starter`) — механизм автоконфигурации Spring Boot, который автоматически настраивает компоненты приложения на основе classpath и свойств `application.properties`.
